# ADR-003 - Gameplay Tags Strategy

Status: Accepted

Date: 2026-07-01

---

# Context

Gameplay Tags can quickly become difficult to maintain if every possible tag is created from the beginning.

---

# Decision

Gameplay Tags are introduced incrementally.

Only tags required by the current framework sprint are created.

---

# Rationale

Avoid unused tags.

Keep the hierarchy clean.

Reduce maintenance.

Allow the tag library to grow naturally.

---

# Consequences

Sprint 2 introduces only item-related tags.

Future sprints extend the hierarchy when required.
