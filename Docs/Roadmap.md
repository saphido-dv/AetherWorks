# Sprint Overview

The development of Aetherworks is organized into structured sprints.

Each sprint represents a coherent framework milestone.

No sprint is skipped.

No gameplay system is implemented outside of a sprint.

---

# Long Term Vision

Aetherworks aims to become a modular, scalable and reusable multiplayer factory framework.

Framework systems remain generic.

Gameplay systems consume framework systems.

New gameplay should not require framework modifications.

---

# Rules

- No sprint skipping
- No feature outside sprint scope
- Documentation first
- Multiplayer first
- Data driven
- Event driven
- Framework before gameplay
- Framework Ready before moving to the next sprint
